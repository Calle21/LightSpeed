Basic syntax

Nova has a similar syntax to C, albeit cleaner and more flexible. Here is an example of a basic function definition.

                Int biggestoftwo Int a Int b
                  a > b ? a
                  b

All typenames begin with an upper-case letter, followed by a string of lower-case, upper-case and numbers. The regexp would look like this

[A-Z][a-zA-Z0-9]*

And additionally, the string must contain at least one number or lower-case letter. Symbols consisting of just upper-case letters are interpreted as type-variables, which we will get into later.

Function names consist of a string of lower-case and numbers. The regexp would look like this

[a-z0-9]+

Where at least one character is a letter. Symbols containing just digits are interpreted as numbers

If statements are made up of a predicate followed by a question mark. If the predicate returns true, the code after the question mark is executed. Otherwise execution continues to the next statement.

As seen above, the return type of a function is written first, followed by the function name, which is followed by a parameter list. The parameter list does not include parentheses or commas, to improve readablility and minimize bugs.

                Bool isfunctionname String s
                  all (lower || digit) s & some lower s

Above is a demonstration of a tuple. Tuples are made up of parentheses, with expressions within them. In this case there is only one expression within the tuple, a predicate, and thus the tuple is only used for evaluation precedence purposes.
